Conectar con Google, paso a paso

Para compartir un viaje, la app necesita tres valores de Google Cloud. Se consiguen gratis, una sola vez, y son públicos (no son contraseñas): Client ID, API key y número de proyecto. Guía vigente para la consola de 2026, donde la configuración vive en «Google Auth Platform».

1Crear el proyecto

Entrá a console.cloud.google.com con tu cuenta de Google. Arriba a la izquierda, tocá el selector de proyectos → Proyecto nuevo. Nombre sugerido: travel-42uy. Crealo y asegurate de que quede seleccionado.

2Anotar el número de proyecto

En el panel de inicio (Descripción general de Cloud → Panel), la tarjeta Información del proyecto muestra Número del proyecto (solo dígitos, p. ej. 512874063921). También está en ☰ → IAM y administración → Configuración.

→ Este es el «Número de proyecto» que pide la app.

3Habilitar las dos APIs

☰ → APIs y servicios → Biblioteca. Buscá y habilitá, una por una:

  • Google Drive API
  • Google Picker API

4Configurar Google Auth Platform

☰ → Google Auth Platform (si no aparece, buscá «Google Auth Platform» en la barra de búsqueda de la consola). La primera vez muestra «Google Auth Platform no está configurado» → tocá Comenzar. El asistente pide:

  • Información de la app: nombre (p. ej. «Travel») y tu email de asistencia.
  • Público (Audience): elegí Externo. No se puede cambiar después.
  • Información de contacto: tu email.
  • Aceptá la política y tocá Crear.
La app queda en modo Prueba (Testing): solo las cuentas que agregues como «test users» van a poder conectarse. Para un grupo de amigos alcanza y no hace falta el proceso de verificación de Google (hasta 100 usuarios).

5Agregar a tus amigos como usuarios de prueba

Dentro de Google Auth Platform, andá a la página Público (Audience). En la sección Usuarios de prueba tocá + Add users y agregá tu email y el de cada persona que vayas a invitar a un viaje.

Si alguien ve «Error 403: access_denied» al conectar, es esto: su email no está en la lista de usuarios de prueba.

6Crear el Client ID

Dentro de Google Auth Platform, página Clientes (Clients) → + Crear cliente:

  • Tipo de aplicación: Aplicación web.
  • Nombre: «Travel PWA» (o el que quieras).
  • Orígenes de JavaScript autorizados — agregá los dos: https://travel.42.uy http://localhost:8765
  • URIs de redireccionamiento: dejar vacío (no se usa).

Tocá Crear y copiá el ID de cliente.

→ Este es el «Client ID»: termina en .apps.googleusercontent.com.

7Crear la API key

☰ → APIs y servicios → Credenciales → + Crear credenciales → Clave de API. Copiá la clave (empieza con AIza).

Recomendado: tocá la clave para editarla y restringila — Restricciones de aplicaciones → Sitios web con los mismos dos orígenes del paso 6, y Restricciones de API → Google Picker API.

→ Esta es la «API key».

8Pegar los tres valores en la app

Volvé a la app → Herramientas → Compartir viaje. La primera vez, el formulario «Conectar con Google» pide los tres valores; si ya lo cerraste, está en Configuración de Google… al pie del diálogo. Se guardan solo en tu dispositivo.

Al conectar por primera vez Google muestra «Google no verificó esta app»: es lo esperado en modo Prueba. Tocá Continuar. La autorización real la controlan los permisos de la carpeta compartida en Drive.
Los tres valores son identificadores públicos. La app usa el permiso drive.file: solo ve las carpetas y archivos que ella misma crea o que vos elegís explícitamente.
